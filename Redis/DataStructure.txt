1.SDS字符串 
1.1SDS字符串结构
struct SDS {
    int length;
    int free;
    char buf[];
}
2.SDS字符串特点
常数级别获取字符串长度
杜绝缓冲区溢出
减少内存重分配 
    1.空间预分配    每次对SDS字符串修改，需要扩展字符串大小时，在字符串小于1M的情况下，
会预留出相当于length长度的空间，在字符串大于1M的情况下，会预留出1M的空间
    2.惰性空间释放    当对SDS字符串修改时，需要缩小字符串时，并不会立即释放多余的空间，而是将多余空间的长度
保存在free字段里，当需要增长字符串长度时就不需要额外分配空间了，SDS操作API也为我们提供了释放空间的方法防止内存泄漏
二进制安全
    SDS字符串并不是通过字符存储的，而是存储的二进制字节
兼容部分C语言字符串处理函数
    SDS字符串在末尾也添加了空字符结尾，从而可以重用一部分C语言库字符函数
